# Task 1 – API для управления задачами

## Задание

Реализовать API для управления списком задач:
- **Создание задачи** (`POST /tasks`)
- **Получение всех задач** (`GET /tasks`)
- **Получение задачи по ID** (`GET /tasks/{task_id}`)
- **Изменение задачи** (`PUT /tasks/{task_id}`)
- **Удаление задачи** (`DELETE /tasks/{task_id}`)

### Требования:
- Использование **Pydantic-схем** (`TaskCreate`, `TaskUpdate`, `Task`).
- Дополнительная **валидация**: для поля `title` установлена длина 3–100 символов и запрет на ввод только пробелов.
- Соблюдение **архитектуры слоев**: разделение на `api`, `schemas` и `services`.
- Хранение данных: в оперативной памяти (Python list).

## Структура проекта

- `main.py` – точка входа: инициализация FastAPI и подключение роутеров.
- `api/tasks.py` – описание HTTP-эндпоинтов.
- `schemas/tasks.py` – Pydantic-модели и правила валидации данных.
- `services/tasks.py` – бизнес-логика (класс `TaskService`) и хранилище задач.
- `__init__.py` – маркеры пакетов в каждой директории.

## Запуск проекта

Из корневой папки проекта выполните:

```bash
uvicorn main:app --reload
```

## Запуск и проверка

**После запуска:**
- **Swagger UI** (интерактивная документация): [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)
- **Проверка работоспособности**: `GET /` → `{ "message": "Task management API is running" }`

### Инструкция по тестированию:

1. **Запустите сервер** и откройте Swagger в браузере.
2. **Создайте задачу** через `POST /tasks/`:
   - Нажмите **Try it out**.
   - Используйте пример данных:
     ```json
     {
       "title": "Сделать лабу",
       "description": "Реализовать API на FastAPI",
       "is_done": false
     }
     ```
   - Нажмите **Execute**. Статус `201` подтверждает успех.
3. **Проверьте валидацию**: попробуйте отправить заголовок `"   "` (только пробелы). Вы должны получить ошибку `422 Unprocessable Entity`.
4. **Просмотрите список**: выполните `GET /tasks/`, чтобы увидеть созданную задачу.
5. **Обновите статус**: через `PUT /tasks/{task_id}` измените `is_done` на `true`.
6. **Удалите задачу**: выполните `DELETE /tasks/{task_id}`. Повторный поиск этой задачи вернет `404 Not Found`.
