# Задание 3: Продвинутое API для управления задачами

## Описание проекта
В данной части работы проект переведен на многослойную архитектуру (Clean Architecture) и дополнен функционалом комментариев, версионированием и автоматизированным тестированием.

## Реализованный функционал:

### 1. Сущность Комментарии
- Реализована связь One-to-Many между задачами и комментариями.
- Созданы эндпоинты для добавления и получения комментариев, привязанных к конкретной задаче.

### 2. Версионирование API (v1)
- Все маршруты сгруппированы под префиксом /v1/.
- Структура роутеров разделена на модули: `auth`, `tasks` и `comments`.

### 3. Глобальная обработка ошибок
- Реализован кастомный класс исключений `AppException`.
- Настроен хендлер, возвращающий ошибки в едином формате `JSON`:
  `{
    "error": {
      "code": "TASK_NOT_FOUND",
      "message": "Задача не найдена"
    }
  }`

### 4. Тестирование (Pytest)
- Unit-тесты: Проверка логики `TaskService` с использованием `AsyncMock` для изоляции от базы данных.
- Интеграционные тесты: Проверка защиты эндпоинтов и корректности статус-кодов через `httpx.AsyncClient` и `ASGITransport`.

## Структура проекта
- `app/repositories/` - Слой работы с базой данных (SQLAlchemy).
- `app/routers/` - Эндпоинты API (FastAPI).
- `app/services/` - Бизнес-логика и валидация.
- `app/models.py` - Описание таблиц базы данных.
- `app/schemas.py` - Pydantic модели для валидации данных.
- `app/exceptions.py` - Кастомные ошибки и хендлеры.
- `app/database.py` - Настройка асинхронного подключения к БД.
- `app/main.py` - Точка входа в приложение.
- `tests/` - Автоматические тесты (Unit и Integration).

## Запуск и проверка

### Установка зависимостей:
```bash
pip install -r requirements.txt
```
### Запуск сервера:
```bash
python -m uvicorn app.main:app --reload
```
### Запуск тестов:
```bash
python -m pytest
```
## Инструкция по ручному тестированию:

1. Авторизация: Зарегистрируйтесь в `/auth/register` и получите токен в `/auth/login`. Используйте кнопку Authorize в Swagger для вставки Bearer-токена.
2. Задачи: Создайте задачу через POST `/v1/tasks/`. Сохраните ID созданной задачи.
3. Комментарии: Добавьте текст комментария через POST `/v1/tasks/{task_id}/comments`.
4. Ошибки: Запросите задачу с несуществующим ID (например, 999), чтобы проверить формат ответа `TASK_NOT_FOUND`.
