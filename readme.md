# Задание 2 - API для управления задачами (DB + Auth)

## Задание

Реализовать продвинутую версию API для управления списком задач с использованием базы данных и авторизации:
- **Регистрация пользователей** (`POST /auth/register`)
- **Вход и получение токена** (`POST /auth/login`)
- **Создание задачи с привязкой к юзеру** (`POST /tasks`)
- **Получение своих задач** (`GET /tasks`)
- **Удаление своих задач** (`DELETE /tasks/{task_id}`)

### Требования:
- **Хранение данных**: Использование **SQLAlchemy** и асинхронного драйвера `aiosqlite` для работы с базой **SQLite**.
- **Безопасность**: Реализована **JWT-авторизация** и хеширование паролей через `bcrypt`.
- **Архитектура слоев**: Продвинутое разделение на `routers`, `schemas`, `services` и `repositories`.
- **Приватность**: Пользователь имеет доступ только к тем задачам, которые создал сам (`owner_id`).

## Структура проекта

Весь код вынесен в пакет `app/` для лучшей изоляции:
- `app/main.py` – точка входа и подключение роутеров.
- `app/routers/` – эндпоинты для задач и авторизации.
- `app/services/` – бизнес-логика (связующее звено).
- `app/repositories/` – прямой доступ к БД (SQL-запросы через ORM).
- `app/models.py` – описание таблиц базы данных.
- `app/schemas.py` – Pydantic-модели для валидации.
- `app/auth.py` – логика работы с токенами и паролями.
- `app/database.py` – настройки подключения к асинхронной БД.

## Запуск проекта

Убедитесь, что вы находитесь в корневой директории проекта, и выполните:

```bash
python -m uvicorn app.main:app --reload 
```
## Запуск и проверка

**После запуска:**
- **Swagger UI** (интерактивная документация): [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)
- **База данных**: Файл `sql_app.db` создастся автоматически при первом запуске приложения.

### Инструкция по тестированию:

1. **Зарегистрируйтесь**:
   - Откройте метод `POST /auth/register`.
   - Нажмите **Try it out** и создайте пользователя (логин и пароль).
   - Нажмите **Execute**. Статус `200` подтверждает успех.

2. **Авторизуйтесь**:
   - Нажмите кнопку **Authorize** (зеленый замок) в самом верху страницы.
   - Введите логин и пароль созданного пользователя.
   - Нажмите **Authorize**, затем **Close**. Теперь вы авторизованы.

3. **Создайте задачу**:
   - В методе `POST /tasks/` добавьте новую задачу.
   - В ответе убедитесь, что поле `owner_id` соответствует вашему ID (обычно это 1 или 2).

4. **Просмотрите список**:
   - Выполните `GET /tasks/`. Вы должны увидеть только те задачи, которые создали сами.

5. **Проверьте защиту**:
   - Поднимитесь вверх страницы, нажмите **Authorize** -> **Logout**.
   - Попробуйте снова выполнить `GET /tasks/`.
   - Сервер должен вернуть ошибку `401 Unauthorized`, так как доступ без токена запрещен.
