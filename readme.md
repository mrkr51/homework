# Задание 4: Docker, MinIO (S3) и Асинхронная архитектура

## Описание проекта
В данной части работы проект был полностью переведен на асинхронную архитектуру, упакован в Docker-контейнеры и интегрирован с облачным S3-совместимым хранилищем (MinIO) для работы с файлами.

## Реализованный функционал:

### 1. Асинхронность и CORS
- Все эндпоинты, сервисы и репозитории переведены на `async/await`.
- База данных работает через асинхронный драйвер `aiosqlite`.
- Настроен `CORSMiddleware` для разрешения кросс-доменных запросов (CORS разрешен для всех origin в рамках dev-окружения).

### 2. Работа с файлами (MinIO / S3)
- Поднято локальное S3-хранилище через образ MinIO.
- Интегрирована асинхронная библиотека `aioboto3` для взаимодействия с хранилищем.
- Реализован эндпоинт `POST /v1/tasks/{task_id}/upload-avatar` для загрузки изображений. Эндпоинт сохраняет файл в корзину `avatars` и возвращает публичный URL файла.

### 3. Системные эндпоинты
- `GET /health` — проверка состояния приложения. Выполняет тестовый запрос к БД и пингует S3-хранилище. Возвращает статус компонентов.
- `GET /info` — отдает информацию о версии приложения и окружении.

### 4. Контейнеризация (Docker)
- Написан `Dockerfile` для сборки легковесного образа на базе `python:3.12-slim`.
- Создан `docker-compose.yml` для одновременного запуска приложения и инфраструктуры (базы данных и MinIO) одной командой.

## Структура проекта
- app/repositories/ - Слой работы с БД.
- app/routers/ - Эндпоинты API (auth, tasks, comments, system).
- app/services/ - Бизнес-логика.
- app/utils/ - Вспомогательные утилиты (s3.py для работы с хранилищем).
- app/config.py - Чтение переменных окружения.
- tests/ - Авто

### Доступные сервисы после запуска

* **API (Swagger):** [http://localhost:8000/docs](http://localhost:8000/docs)
* **Health Check:** [http://localhost:8000/health](http://localhost:8000/health)
* **MinIO Console (Панель управления S3):** [http://localhost:9001](http://localhost:9001) 
  * *Логин / Пароль:* `minioadmin` / `minioadmin`

### Инструкция по проверке загрузки файлов

1. Зайдите в панель управления **MinIO** ([http://localhost:9001](http://localhost:9001)).
2. Создайте корзину (Bucket) с названием `avatars` и установите для нее **Access Policy** -> **Public**.
3. Авторизуйтесь в **Swagger API**.
4. Выполните запрос `POST` к `/v1/tasks/{task_id}/upload-avatar`, прикрепив изображение.
5. Перейдите по URL, полученному в ответе, чтобы убедиться, что загруженный файл корректно открывается в браузере.
